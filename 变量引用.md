## 变量问题

刚开始学习python的时候总是容易碰见一些变量引用上出现的问题，而经常会听到诸如值传递和引用传递这种概念，而其实弄懂了底层的实现原理后本质上就不会有值传递和引用传递这种毫无意义的概念。

首选要了解python的数据类型有诸如：数值（int、float）、字符串(str)、列表(list)、集合(set)、字典(dict)、元祖(tuple)、布尔值(bool)。这些数据类型可分为两大类：可变与不可变。其中不可变指的是：数值、字符串、元祖、布尔值。其他的是可变数据类型。python中创建一个变量，如 i =1 或I=[]则会在内存地址中分配一块地址来存储1或[]，而变量i则指向了该地址。而不可变变量指的是，其在内存地址中是共享的，比如创建j = 1，该j指向1对应的内存地址，与 i中1的内存地址相同指向同一个，也就是说不可变变量中相同的值在python解释器中是单例的指向相同内存地址，而可变变量刚好相反，比如创建对象k =[],虽然与i=[]相同，但是其内存地址完全不相同。

上面讲解了可变与不可变的在python中内存的区别，这次来讲解一下，python中对变量引用的实现原理。首先先抛开可变与不可变的概念，创建变量i = 1, 然后将i赋值给j,也就是j=i，此时python中发生了什么呢？其实是将j直接指向了1对应的内存地址，记住不知将j指向i，好了到这里我们重新带入上面可变变量和不可变变量的概念，接着我们将i又赋值为2,此时j会跟着变成2吗，答案当然是否定了，因为j一开始就是指向1的变量地址，除非对该变量地址上的值改变，j才会改变，而之前也提到过了1是不可变变量，对其进行改变地址也会发生改变。ok，至此，不可变变量的引用传递问题迎刃而解。接下来就是可变变量了，重新创建一个变量li =[] ,然后li2 = li ,接着对li.append(1),此时不可变变量的内存地址不会改变，所以li2 ==[1];如果li = 1 ,li2会变成1,答案当然是no了，因为此时只是将li指向了1的内存地址，而[1]对应的内存地址不变，li2还是指向的[1]的内存地址。

总结可变与不可变是指变量修改后内存地址上的可变与不可变;变量引用是对内存地址的引用，想要改变变量值需要改变的是内存地址上的值。

j = 1
id(i) == id(j)
>>>>>>>>
i = []
j = []
id(i) != id(j)
>>>>>>>>
i = 1
j = i 
i = 2
j == 1
>>>>>>
i = []

i.append(1)

>>>>>>
